One-Step Install

If you are on linux, you can run the following command to install in one step in a directory where you want the repository to be cloned: 
$ bash <(curl -L -s http://pkambadu.github.io/AMD/install_amd.sh)
If this does not work, you can follow the commands below to install AMD.

Download

To download the latest version of AMD, please run the command: 
$ git clone http://github.com/pkambadu/AMD.git

Installation

AMD is configured, compiled, and installed using the Python-based Waf* program.

Why Waf?

The following quote on the Black Duck Open Hub review of Waf** best describes the reason why we decided to choose Waf:
  • Waf depends on Python only which is ported on most platforms
  • Waf scripts are Python modules which are easier to learn and to maintain that custom languages
  • Waf license has very little constraints (BSD) and can be redistributed easily (all in one 100kb script)
  • Waf architecture is modular and can be extended easily, it relies on state-of-the-art algorithms
  • Waf provides many more features than its competitors
  • Waf provides many small projects and code snippets

System Requirements

  • git
  • Python 2.6 or 2.7
  • Python C libraries (e.g., libpython2.7)
  • C and C++ compilers supporting C++03 standard.
  • GNU make: make in Linux, or gnumake in darwin-MacOSX
  • Boost***

Configure

After download, cd to the top-level directory, then run one of the following commands:
  • To build the libraries and the examples, run: 
    $ ./waf configure
  • To build the libraries only, run: 
    $ ./waf configure --bld-AMD
  • To build the examples, run: 
    $ ./waf configure --bld-examples
  • To build the tests, run: 
    $ ./waf configure --bld-AMD --tests
  • To build Python bindings, run 
    $ ./waf configure --bld-python
  • Individual examples, packages, and dependencies can also be configured to be built, and command line flags can be combined. Run the following command for more information about available options: 
    $ ./waf --help

Build

To actually execute the build after running one of the configuration commands above, run 
$ ./waf build
See "Troubleshooting" below for possible solutions in case the build command above does not succeed.

Installing Packages

Most of the required third-party dependencies can be automatically installed as follows. After downloading NLP<Go>, cd to the top-level directory and run the command: 
$ ./waf configure --install-deps

Running AMD

To run AMD, run the following command: 
./build/examples/spirit/parser/parser
and enter your desired expression. Examples of valid expressions can be viewed on the examples page.

Documentation

Examples of using the framework and provided attribute and algorithm classes can be found in the examples directory. Every header file has complete documentation of the public classes contained in that component.

Troubleshooting

If Waf cannot find python lib

If Waf cannot find the Python lib, i.e. a message “/usr/bin/ld: cannot find -lpython” is obtained after trying to build. A possible solution is to create a symbolic link to the library in /usr/lib/. For example, run the command 
$ sudo ln -s /usr/lib/x86_64-linux-gnu/libpython2.7.so.1.0 /usr/lib/libpython2.7.so
